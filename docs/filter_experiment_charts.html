實驗洞察

  • Oracle filter 有效：候選池從 2,795 縮至 808.7（-71%），R@10 從 0.2431 → 0.3722（+14.9pp）
  • P1 regex filter 反效果：R@10 0.2222（-2.1pp vs no_filter），因 ExactMatch 63.2% 不夠高，parser 誤解導致 gold 被 filter 掉
  • Oracle R@10 上界低（0.37）的原因：eval_conditions_v1 設計為 filter-hard 題型（如「找星期二的課」），query 缺乏語意線索，dense retrieval 接近隨機猜
  • 速度：oracle（54ms）比 no_filter（499ms）快 9× ——候選池縮小大幅降低矩陣乘積成本
